# T4：接入 bootstrap 渲染 + esgw bootstrap 命令

- **状态**: 已完成
- **依赖**: T1（config 的 XDS 字段）；与 T3 无代码依赖，可在 T3 后并行或顺序进行

## 目标

实现接入 bootstrap 的确定性渲染纯函数（下发层 §2.7：xds 模式下 Envoy 唯一的 static 文件），并交付 `esgw bootstrap --mode xds` 导出命令（T2/T3 拓扑「仅下发」组合的交付物）。

## 执行步骤

1. `internal/deliver/xds/bootstrap.go`：

   ```go
   // BootstrapOpts 是接入 bootstrap 的渲染参数。
   type BootstrapOpts struct {
       NodeID       string // node.id（默认 esgw-node）
       NodeCluster  string // node.cluster（默认 esgw）
       XDSListen    string // esgw ADS 地址（host:port；loopback 校验归 config 层，这里只做形态解析）
       AdminAddress string // unix:///<path> 或 host:port（SD5）
   }
   func RenderBootstrap(opts BootstrapOpts) ([]byte, error) // 纯函数，确定性
   ```

   内容按 §2.7 骨架：`node`、`admin`（pipe 或 socket_address）、`dynamic_resources`（ads_config: GRPC/V3 + cds_config/lds_config 走 ads）、`static_resources.clusters[esgw_xds]`（STATIC、explicit_http_config h2、load_assignment 指向 XDSListen）。文件头注释风格与 static.Render 一致（`# Generated by esgw. DO NOT EDIT.`；不写 config_version——bootstrap 与配置版本无关，写 `node_id` 即可，取舍记进展记录）。
   - 构建方式建议：直接构造 `bootstrapv3.Bootstrap` proto → protojson→map→复用 static 包的确定性 YAML 发射器（若发射器未导出则提取为 internal 共享小函数，改动最小化）。产物经 PGV Validate 自检。
2. `cmd/esgw/bootstrap.go`：`esgw bootstrap -c <esgw.yaml> [--mode xds] -o <file>`——从 esgw.yaml 取 nodeID/nodeCluster/listen/adminAddress 渲染导出；`--mode` 当前仅接受 `xds`（static 无接入 bootstrap 概念，传 static 报用法错误 exit 2）。输出契约同 compile（-o 缺省 stdout）。
3. 测试：
   - 单测：默认值渲染、unix/-tcp 两种 admin 形态、确定性（两次渲染字节级一致）、非法 AdminAddress 报错；产物关键字段断言（ads_config、esgw_xds cluster 地址、h2 显式开启）。
   - golden（可选，推荐）：`testdata/bootstrap-xds/golden.yaml` 快照，纳入 internal/golden 体系或包内 golden，沿用 `-update` 约定。
   - 产物经 `envoy --mode validate` 验（包内测试复用 envoycheck；无二进制时按既有约定降级 skip/warning）。
4. `make build test lint` 全绿。commit scope：`xds`（RenderBootstrap）+ `cmd`（bootstrap 子命令）。

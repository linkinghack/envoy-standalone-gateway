## Sprints敏捷项目冲刺迭代-文档驱动项目进展记录

当前目录中应该按照每个敏捷冲刺创建子文件夹，以冲刺开始日期作为冲刺编号，每个冲刺中应该包含一下内容：
1. `requirements.md` 描述本次冲刺要解决的问题（需求）内容，要达成的目标和验收标准
2. `technical_design.md` 针对本次需求进行的系统详细设计，包括关键的架构决策、技术方案设计、关键流程设计等
    注意：对于需要沉淀下来在后续的开发过程中不断参考的内容（如API文档、系统领域模型设计文档、技术方向决策参考等类似的内容，应该放在 ../dev_design目录下，不属于Sprints目录承载的内容）
3.  `plan_todos_trace.md` 用于针对本次冲刺进行任务拆分并追踪开发进展，便于指导Codex或ClaudeCode在多次新会话中快速进入正在进行的工作，在开发过程中必须及时更新此进展。
4. `extra_docs/` 用于补充多文件设计文档或其他需要补充说明的内容
5. 针对每个Sprint, 内部可能规划多个task，每个task应该是一个相对独立的开发过程，针对每个task，应该给出一个详细的执行步骤并在开发过程中不断更新记录。在每个sprint目录内创建tasks/ 目录，并在这里以task ID为名创建task todo跟踪文档。

冲刺总体规划与里程碑对应关系见 [`../260719-1-dev-roadmap.md`](../260719-1-dev-roadmap.md)。

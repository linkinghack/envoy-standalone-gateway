# T6 static 渲染 + F7 envoycheck + esgw compile CLI

- **状态**: 已完成
- **依赖**: T5
- **设计依据**: [下发层 §4.1 渲染契约](../../../../system_design/260717-1-deliver-layer-design.md)、[编译层 §3/F7、§6 CLI](../../../../system_design/260716-2-compile-ir-design.md)、[状态采集 §4.1 版本注入契约](../../../../system_design/260717-4-state-collection-design.md)

## 目标

IR → 单文件 static YAML 渲染（纯函数）、`envoy --mode validate` 封装、面向用户的第一个交付物 `esgw compile`。

## 执行步骤

1. **`internal/deliver/static.Render(ir)`**：输出单文件 envoy.yaml——文件头注释（`Generated by esgw` + `config_version`）；`node.metadata` 注入 `esgw.config_version: <IR.Version>`（渲染期注入、不回改 IR，260717-4 §4.1 契约）；admin 段 M0 默认 uds 路径；proto→YAML 经 protojson→map→yaml 确定性输出（key 排序）。
2. **`internal/compile/envoycheck`**：发现 envoy 二进制（`ESGW_ENVOY_PATH` → PATH）；`envoy --mode validate -c <tmpfile>` exec 封装，超时与 stderr 捕获；无二进制返回 Warning 级 CompileError（编译层 F7 语义）；独立子包不被编译器主体 import（SD5）。
3. **CLI `cmd/esgw`**（框架用 spf13/cobra 或标准库 flag——依赖最小优先，M0 只有一个子命令，倾向标准库 + 简单分发，S2 增加 serve/bootstrap 时再评估）：
   - `esgw compile -f <dir> --mode static|xds -o <file>`：LoadDir → Compile → static 渲染或 xds snapshot protojson 输出；
   - `--envoy-validate[=path]` 开关触发 F7；
   - 错误输出：LoadError/CompileError 按 `file:kind/name:path: message` 逐行打印，Error 存在时 exit code 1，Warning 仅提示；
   - `esgw version`。
4. **CLI 集成测试**：以 testdata 目录跑 compile 两种 mode，断言产物文件与 exit code；错误目录断言错误行格式。

## 验收

- [x] `esgw compile -f testdata/s1/input --mode static -o /tmp/envoy.yaml` 产出可读产物且含版本标记（本任务用 `cmd/esgw/testdata/ok`（无证书依赖）验收；testdata/s1 属 T7 golden 用例，届时同一 CLI 直接复用）
- [x] `--mode xds` 输出 snapshot protojson
- [x] envoycheck 在有/无 envoy 二进制两种环境行为正确（无 = Warning 降级；有 = 假 envoy 脚本覆盖成功/失败两态，见 envoycheck 与 CLI 测试）
- [x] 错误输出格式化且 exit code 语义正确（`error|warning: file:kind/name:path: message`；Error exit 1 / 仅 Warning exit 0 / 用法 exit 2）

### 2026-07-20 T6 完成（commits `ebfbfa7` / `c98b540` / `1ef55de`）

**交付内容**

1. `internal/deliver/static.Render(i *ir.IR) ([]byte, error)`（冻结签名，纯函数）：
   - 文件头 `# Generated by esgw. DO NOT EDIT.` + `# config_version: <IR.Version>`；
   - `node.metadata` 渲染期注入 `esgw.config_version: <IR.Version>`——在深拷贝的 Bootstrap 上注入，不回改 IR、不参与版本哈希（260717-4 §4.1），`TestRenderDoesNotMutateIR` 锁定；
   - admin 段渲染期覆盖为 M0 默认 uds `/tmp/esgw-admin.sock`（常量 `DefaultAdminSocketPath`）；
   - proto→YAML 链路：protojson（UseProtoNames，snake_case 与 Envoy 文档一致）→ `map[string]any` → 自研确定性 YAML 发射器（map 键字典序、标量/键的引用风格由 yaml.v3 单值 Marshal 决定，`@type`、YAML 1.1 布尔词、数字形态字符串均被正确加引号）；`TestRenderDeterministic` 锁定同一 IR 两次渲染字节一致（SD6）。
2. `internal/deliver.SnapshotJSON(i *ir.IR) ([]byte, error)`：xds 模式产物（本任务裁量形态）——单 JSON 文档 `{version, listeners, clusters, routes, endpoints, secrets}`，每类资源为 `{<资源名>: <protojson>}` map；encoding/json map 键字典序 + 固定两空格缩进保证确定性；Bootstrap 不在其中（接入 bootstrap 是独立产物，下发层 §2.7，T7/S2 再做）。S2 ADS server 消费时可调整。
3. `internal/compile/envoycheck`（纯 stdlib，编译器主体不 import，SD5）：
   - `FindBinary(explicit)`：显式路径 → `ESGW_ENVOY_PATH` → PATH `envoy`，全部落空返回包装 `ErrBinaryNotFound` 的错误；
   - `Validate(ctx, binPath, config, timeout)`：写临时文件 → `envoy --mode validate -c <tmp>`，默认 30s 超时（context 控制并明确报告 timeout），失败带回 stderr 尾部 ≤4KB；本包只报告事实，Warning/Error 分级由调用方（CLI）包装为 Stage=envoy 的 CompileError。
4. CLI `cmd/esgw`（标准库 flag + 简单分发，未引 cobra）：
   - `esgw compile -f <dir> --mode static|xds -o <file> [--envoy-validate[=<path>]]`；`-o` 缺省写 stdout；`--mode` 默认 `static`；
   - `--envoy-validate` 支持可选值（预扫描剥离，路径只接受 `=` 形式）；F7 由 CLI 层编排：xds 模式重新加载目录并以 ModeStatic 二次编译渲染等价 static 再验（编译层 §3 F7 语义；Compile 会原地填充默认值故必须重新 LoadDir）；
   - F7 失败（envoy 拒绝）→ Error 级、exit 1、**不写产物**；无二进制 → Warning 级、exit 0、产物照写；
   - 错误逐行 stderr：`error|warning: file:kind/name:path: message`（LoadError 无 kind/name/path，输出 `error: file: message`；无文件来源标 `<implicit>`）；
   - `esgw version` 保留；`main` 重构为 `run(args, stdout, stderr) int` 便于集成测试断言退出码。
5. 测试：static 渲染 7 例、SnapshotJSON、envoycheck 7 例（假 envoy 脚本）、CLI 集成 11 例（两 mode 产物与 exit code、错误行格式逐字断言、F7 三态含"xds 模式渲染 static 后验"）；`gofumpt -w .` + `make build && make test && make lint` 全绿（0 issues）。

**实现决策 / 与设计的偏离（无静默偏离）**

1. **admin uds 路径取 `/tmp/esgw-admin.sock` 而非下发层 §4.1 示例的 `/var/run/esgw/envoy-admin.sock`**：M0 CLI 是纯导出场景，无数据目录概念，写 `/var/run/esgw/` 对多数环境不可写；`/var/run/esgw/...` 形态留给 S7 M-PROC 托管落盘（数据目录就位后由 writer/托管层使用）。
2. **T5 Bootstrap 骨架的 admin（127.0.0.1:9901）在渲染期被覆盖为 uds**（任务书明示"渲染期可覆盖/补齐"；骨架 admin 仅为占位）。`node.id/cluster`（esgw/esgw）保持骨架值不变；下发层 §2.3 的 `esgw-node`/`esgw` 约定属 xds 接入 bootstrap 范畴，S2 接线 ADS 时统一。
3. **`Options.EnvoyValidate` 字段 M0 不在库内消费**：F7 由 CLI 层编排（编译层 §3"不放进库内同步路径"），`Compile()` 保持纯函数，冻结签名未动。该字段保留作为 M1+ 发布流注入点。
4. **static 模式残留的 IR.Routes/IR.Endpoints 不输出**：Bootstrap.StaticResources 只有 listeners/clusters/secrets 字段；F5 已内联编译产物的 route_config 与端点，残留项只可能是 EnvoyResources 引入的引用形态资源（RDS/EDS 引用在 static bootstrap 下本就不成立）。Render 选择不输出而非报错——该情形的拦截归 F6/后续事项，T7 golden 若覆盖此边角再定。
5. **错误行加 `error|warning: ` 前缀**（任务书格式 `file:kind/name:path: message` 的超集）：Warning（F7 缺二进制）与 Error 需要在输出上可区分，不能仅靠 exit code。
6. **`--mode` 默认 `static`**：CLI 定位是"拿走 YAML 脱离 esgw 使用"的导出路径（下发层 §1.2），static 是该路径的主产物；运行期默认 xds 的决策（下发层 §1.2）作用于 `deliver.mode` 配置而非 CLI。

- xds 模式接入 bootstrap（dynamic_resources/ADS）导出未做：T5 骨架注释与下发层 §2.7 均指向 S2/T7 之后。
- `esgw compile` 的原子写（tmp+rename）未做：属下发层 writer.go（S7），CLI 直接 WriteFile。
- 真实 envoy 二进制的多版本 validate 矩阵属 T7（`make validate-matrix`）。

- **状态**: 未开始

## 目标

IR → 单文件 static YAML 渲染（纯函数）、`envoy --mode validate` 封装、面向用户的第一个交付物 `esgw compile`。

## 执行步骤

1. **`internal/deliver/static.Render(ir)`**：输出单文件 envoy.yaml——文件头注释（`Generated by esgw` + `config_version`）；`node.metadata` 注入 `esgw.config_version: <IR.Version>`（渲染期注入、不回改 IR，260717-4 §4.1 契约）；admin 段 M0 默认 uds 路径；proto→YAML 经 protojson→map→yaml 确定性输出（key 排序）。
2. **`internal/compile/envoycheck`**：发现 envoy 二进制（`ESGW_ENVOY_PATH` → PATH）；`envoy --mode validate -c <tmpfile>` exec 封装，超时与 stderr 捕获；无二进制返回 Warning 级 CompileError（编译层 F7 语义）；独立子包不被编译器主体 import（SD5）。
3. **CLI `cmd/esgw`**（框架用 spf13/cobra 或标准库 flag——依赖最小优先，M0 只有一个子命令，倾向标准库 + 简单分发，S2 增加 serve/bootstrap 时再评估）：
   - `esgw compile -f <dir> --mode static|xds -o <file>`：LoadDir → Compile → static 渲染或 xds snapshot protojson 输出；
   - `--envoy-validate[=path]` 开关触发 F7；
   - 错误输出：LoadError/CompileError 按 `file:kind/name:path: message` 逐行打印，Error 存在时 exit code 1，Warning 仅提示；
   - `esgw version`。
4. **CLI 集成测试**：以 testdata 目录跑 compile 两种 mode，断言产物文件与 exit code；错误目录断言错误行格式。

## 验收

- [ ] `esgw compile -f testdata/s1/input --mode static -o /tmp/envoy.yaml` 产出可读产物且含版本标记
- [ ] `--mode xds` 输出 snapshot protojson
- [ ] envoycheck 在有/无 envoy 二进制两种环境行为正确（无 = Warning 降级）
- [ ] 错误输出格式化且 exit code 语义正确

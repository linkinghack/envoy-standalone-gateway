# Sprint 260726 需求：交付物与 M1 收口

## 目标

落地路线图 S8，把已完成的 xDS/static、托管/仅下发、管理 API 与控制台包装成可安装、可升级、可备份且可验证的 Linux/Docker 发行形态，并用直接证据关闭 M1 P0 需求矩阵。

## 范围

- systemd unit，安装/升级/卸载脚本，运行用户与目录权限；
- `esgw` 管理面镜像、`esgw-all-in-one` 镜像、compose 示例与健康检查；
- T1/T2 × xDS/static 四组合真实 Envoy e2e；
- 快速开始、配置、运维、备份恢复、升级和安全文档；
- linux/amd64、linux/arm64 发布归档，校验和、许可证清单、SBOM 与 M1 需求证据矩阵。

## 非范围

- deb/rpm、镜像仓库推送与签名基础设施；
- Helm/Kubernetes、跨主机 xDS mTLS、多节点；
- 自动在线备份调度、完整统计/历史 UI 与 P1/P2 能力。

## 验收标准

1. 安装脚本幂等创建 `esgw` 系统用户和 0750/0700/0600 目录/文件，升级保留配置和数据，卸载默认保留数据；
2. systemd 仅停止管理主进程，托管 Envoy/launcher 可跨管理面重启继续服务，低端口能力最小化授予；
3. 两种 OCI 镜像均非 root 运行管理面、带可执行健康检查，compose 可从空环境启动；
4. 四组合运行真实 Envoy，验证健康、流量、配置生效边界与管理面故障不影响最后有效数据面；
5. 快速开始主路径可在 10 分钟内完成，备份/恢复/升级/安全约束明确且命令可执行；
6. 发布脚本生成双架构归档、SHA256、第三方许可证报告和 SPDX JSON SBOM；
7. M1 每条 P0 需求有实现与测试/文档证据，全仓质量门禁通过。
